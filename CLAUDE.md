## 서비스 한 줄 정의

## 핵심 차별화 포인트

기존 WM 서비스는 미래를 단일 숫자로 표현한다.
이 서비스는 미래를 **확률 분포**로 표현한다.

```
기존: "83세까지 버팁니다"
이 서비스: "83세 전 자산 고갈 확률은 41%입니다"
```

대부분의 팀이 제출할 것: LLM + 금융 데이터 → 텍스트 조언
이 서비스가 제출하는 것: 통계 모델 → 확률 기반 인사이트 → LLM이 해석 → 자연어 전달

## 통계 모델 상세

### 1. 생존 분석 (Survival Analysis)
- **목적**: 수명을 단일 숫자가 아닌 확률 분포로 추정
- **방법**: Cox Proportional Hazards Model
- **입력**: 성별, 흡연 여부, 만성질환 유무, BMI
- **데이터**: 통계청 국민생명표
- **라이브러리**: `lifelines`
- **출력**: 개인화된 생존 함수 S(t)
- **Monte Carlo 연결**: 수명 분포에서 샘플링 → 각 시뮬레이션 경로의 종료 시점

### 2. Monte Carlo 시뮬레이션
- **목적**: 자산 고갈 확률과 불확실성 범위 정량화
- **방법**: 10,000회 반복 시뮬레이션
- **확률 변수**:
  - 투자수익률: 역사적 데이터 기반 분포 (KRX)
  - 물가상승률: CPI 시계열 기반 분포 (ECOS)
  - 지출: 로그정규분포
  - 의료비 충격: 포아송 과정
  - 수명: 생존 분석 결과 분포
- **출력**: 자산 고갈 확률, 10/50/90 퍼센타일 경로
- **검증 지표**: Coverage Probability (95% 구간이 실제 95% 포함하는지)

### 3. 연금 최적 인출 전략
- **목적**: 국민연금 수령 시기 최적화
- **방법**: 규칙 기반 시나리오 비교 (수령 시기별 총 기대 수령액 + 손익분기 연령)
- **데이터**: 국민연금공단 통계, 한국은행 ECOS
- **출력**: 시나리오별 총 수령액, 손익분기 연령, 생존 확률 결합 권고

### 4. CVaR 포트폴리오 최적화
- **목적**: 은퇴 후 자산 배분 리스크 관리
- **방법**: Conditional Value at Risk 최소화
- **데이터**: KRX 자산별 수익률/공분산
- **라이브러리**: `cvxpy`
- **출력**: 최적 자산 배분 비율, CVaR 수치

### 5. K-means 군집 분석 + 퍼센타일 랭킹
- **목적**: 유사 조건 또래 대비 재무 상태 비교
- **방법**: K-means 군집화 → 군집 내 퍼센타일 계산
- **군집화 변수**: 나이, 순자산, 소득, 부양가족 수
- **데이터**: 통계청 가계금융복지조사 마이크로데이터 (MDIS)
- **라이브러리**: `scikit-learn`
- **출력**: 동일 군집 내 퍼센타일, 상위 30% 행동 패턴 비교

## 이상치 탐지 (3개 레이어)

### Layer 1: 거래 이상 탐지 (금융사기 방어)
- **방법**: Z-score + IQR + Isolation Forest + CUSUM 조합
- **입력**: 개인 거래 내역 (금액, 시간대, 상대 계좌, 거래 유형)
- **학습**: 비지도 학습 — 개인 기저선 자동 학습 (사전 데이터 불필요)
- **대응**: 이상도 점수 → 낮음(기록)/중간(확인 요청)/높음(거래 보류+알림)
- **보이스피싱 특화**: 처음 거래 계좌 + 비정상 금액 + 단기 연속 이체 조합 감지

### Layer 2: 소비 패턴 이상 탐지 (재무 계획 이탈 경보)
- **방법**: CUSUM + PELT Change Point Detection
- **입력**: 월별 카테고리별 지출
- **기준선**: Monte Carlo 예측 구간 (정상 범위 정의)
- **데이터**: 통계청 가계동향조사 (Prior 설정)
- **출력**: 단발성 이상 vs 구조적 변화 구분

### Layer 3: 생활 패턴 이상 탐지 (독거 고령층 안부 확인)
- **신호 출처**:
  - 카드 결제 패턴 (편의점/마트/약국 결제 중단)
  - AI Agent 대화 응답 빈도 및 패턴
  - 정기 지출 이상 (관리비/통신비 미납)
- **방법**: 개인 활동 기저선 + 복수 신호 점수 합산
- **데이터**: 합성 데이터 (실제 로그 데이터 없으므로)
- **대응**:
  - 점수 누적 → Agent가 먼저 안부 확인 말 걸기
  - 무응답 지속 → 신뢰 연락처 또는 담당 복지사 알림

## 성능 평가 지표

### 통계 모델
| 모델 | 지표 |
|------|------|
| Monte Carlo | Coverage Probability (95% 구간 포함률) |
| 생존 분석 | C-index (0.5~1.0) |
| 이상치 탐지 | F1-score, AUC-ROC (Recall 중시) |
| 군집 분석 | Silhouette Score |

### Agent 행동
| 항목 | 측정 방법 |
|------|---------|
| Tool 선택 정확도 | 테스트 질문 20개 → 올바른 Tool 호출 비율 |
| 파라미터 정확도 | Tool 호출 시 올바른 파라미터 전달 비율 |
| 다중 턴 일관성 | 이전 맥락 파라미터 유지 여부 |
| 수치 전달 정확도 | 통계 모델 출력값 정확 전달 비율 |

### 도메인 특화
| 항목 | 측정 방법 |
|------|---------|
| 언어 가독성 | Flesch Reading Ease 점수 |
| 위험 조언 차단율 | 위험 유도 질문 20개 → 거부/경고 비율 |

## 발표 핵심 메시지

1. **왜 이 서비스인가**: 기존 WM은 결정론적, 이 서비스는 확률론적
2. **무엇이 다른가**: 통계 모델이 계산하고 AI가 설명하는 구조
3. **데이터 근거**: 모든 데이터는 한국 공개 데이터 기반 (생활 패턴 로그만 합성)
4. **검증 가능성**: 평가 지표로 성능 수치 제시
5. **사회적 가치**: 가족 없는 독거 고령층까지 커버하는 포용적 설계
