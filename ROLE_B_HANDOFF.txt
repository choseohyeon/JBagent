4. WMAgent 클래스 사용법 — UI 연결의 핵심
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  from agent.agent import WMAgent

  # 생성 (연령 세그먼트 선택: 1=50-64세 / 2=65-74세 / 3=75세+ / None=자동)
  agent = WMAgent(age_segment=2)

  # 대화 한 턴
  reply = agent.chat("저는 70세이고 자산이 2억원인데 얼마나 버틸까요?")
  print(reply)  # → 자연어 응답

  # 연속 대화 (이전 파라미터 자동 유지됨)
  reply2 = agent.chat("지출을 20만원 줄이면요?")  # 나머지 파라미터 그대로

  # 대화 초기화
  agent.reset()

  ※ agent.chat()은 내부에서 통계 모델을 자동으로 호출합니다.
    UI에서는 오직 agent.chat(user_text)만 호출하면 됩니다.
    통계 모델을 직접 import할 필요 없습니다.

5. 6개 통계 Tool 요약 — Agent가 자동으로 선택
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  (UI 개발자는 직접 호출 불필요. Agent가 판단해서 자동 호출함.)
  (팬 차트 등 직접 시각화가 필요할 때만 아래를 직접 import해서 사용.)

  Tool 1: run_survival
    입력: sex("M"/"F"), current_age, smoking(bool), chronic_disease(bool), bmi
    출력: expected_remaining_life, death_prob_before_80, death_prob_before_90
    사용 시점: "내가 몇 살까지 살까요?", "기대수명이 얼마나요?"

  Tool 2: run_monte_carlo  ← 가장 중요
    입력: sex, current_age, initial_assets(만원), monthly_expense(만원),
          pension_monthly(만원), pension_start_age, stock_ratio(0~1)
    출력: depletion_prob(0~1), median_final_assets(만원),
          asset_paths = {70: {p10, p50, p90}, 75: ..., 80: ..., 85: ..., 90: ...}
    사용 시점: "얼마나 버티나요?", "자산이 고갈될까요?", "나이 들면 얼마 남나요?"
    ★ asset_paths가 팬 차트의 데이터 소스

  Tool 3: run_pension
    입력: current_age, expected_monthly_pension(만원), life_expectancy(년)
    출력: optimal_claim_age, monthly_at_optimal, npv_gain_vs_65, breakeven_age
    사용 시점: "연금을 언제 받는 게 유리한가요?"

  Tool 4: run_portfolio
    입력: risk_level("conservative"/"moderate"/"aggressive")
    출력: weights(KOSPI/KTB 비중), stock_ratio_for_mc, cvar_annual, sharpe_ratio
    사용 시점: "어떻게 투자해야 하나요?", "주식 비율이 얼마가 좋아요?"

  Tool 5: run_clustering
    입력: age, net_assets(만원), income(연간 만원), members(가구원수)
    출력: cluster_id(1~5), net_assets_percentile(0~100), income_percentile,
          top30_avg_assets, top30_avg_income
    사용 시점: "또래랑 비교하면 어떤가요?", "제 재무 상태가 괜찮은가요?"

  Tool 6: run_anomaly_score
    입력: amount(만원), hour(0~23), is_new_account(bool), tx_type, consecutive_count
    출력: score(0~100), level("low"/"medium"/"high"), triggers, recommendation
    사용 시점: 이상 거래 감지, 보이스피싱 탐지

─── 6-3. 채팅창 구현 ──────────────────────────────────────────────────────────

  st.session_state로 Agent 인스턴스와 대화 기록을 유지합니다.

  ┌─────────────────────────────────────────────────────┐
  │  import streamlit as st                              │
  │  from agent.agent import WMAgent                    │
  │                                                      │
  │  # 세션 초기화 (한 번만 실행)                         │
  │  if 'agent' not in st.session_state:                 │
  │      age = st.session_state.get('age', 65)           │
  │      seg = 1 if age < 65 else (2 if age < 75 else 3) │
  │      st.session_state.agent = WMAgent(age_segment=seg)│
  │      st.session_state.messages = []                  │
  │                                                      │
  │  # 대화 기록 표시                                    │
  │  for msg in st.session_state.messages:               │
  │      with st.chat_message(msg['role']):               │
  │          st.write(msg['content'])                    │
  │                                                      │
  │  # 입력창                                            │
  │  if user_input := st.chat_input("무엇이든 물어보세요"):│
  │      st.session_state.messages.append(               │
  │          {'role': 'user', 'content': user_input})    │
  │      with st.spinner('계산 중...'):                   │
  │          reply = st.session_state.agent.chat(        │
  │              user_input)                             │
  │      st.session_state.messages.append(               │
  │          {'role': 'assistant', 'content': reply})    │
  │      st.rerun()                                      │
  └─────────────────────────────────────────────────────┘

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  총 20개 테스트. 각 케이스는 (입력 질문, 기대 Tool 호출, 검증 기준)으로 구성.

  ┌─────────────────────────────────────────────────────┐
  │  from agent.agent import WMAgent                    │
  │  import json                                         │
  │                                                      │
  │  TEST_CASES = [                                      │
  │      # (질문, 기대_tool, 검증_키)                     │
  │      ("65세 남성인데 80세 전 사망 확률이 얼마나요?",    │
  │       "run_survival", "death_prob_before_80"),        │
  │      ...                                             │
  │  ]                                                   │
  │                                                      │
  │  def run_all():                                      │
  │      results = []                                    │
  │      for q, expected_tool, check_key in TEST_CASES:  │
  │          agent = WMAgent()                           │
  │          reply = agent.chat(q)                       │
  │          # messages에서 tool_use 블록 검색            │
  │          tool_called = any(                          │
  │              b.type == 'tool_use' and                │
  │              b.name == expected_tool                 │
  │              for m in agent.messages                 │
  │              for b in (m['content']                  │
  │              if isinstance(m['content'], list) else [])│
  │          )                                          │
  │          results.append({                            │
  │              'question': q,                          │
  │              'tool_correct': tool_called,            │
  │              'reply': reply,                         │
  │          })                                          │
  │      return results                                  │
  └─────────────────────────────────────────────────────┘

8. 시스템 프롬프트 구조 — 수정 방법
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  agent/prompts.py의 build_system_prompt(age_segment) 함수가
  연령에 맞는 시스템 프롬프트를 생성합니다.

  세그먼트 1 (50~64세): 일반 성인 수준, 은퇴 준비 중심
  세그먼트 2 (65~74세): 쉬운 단어, 핵심 수치 강조
  세그먼트 3 (75세+):  매우 따뜻하고 간결, 핵심 하나만

  UI에서 나이 입력 시 아래 로직으로 세그먼트를 자동 결정하세요:

    if age < 65:    seg = 1
    elif age < 75:  seg = 2
    else:           seg = 3

    agent = WMAgent(age_segment=seg)

  프롬프트를 수정하고 싶다면 agent/prompts.py의 _SEGMENT_GUIDE,
  _LANGUAGE_GUIDE 딕셔너리를 직접 수정하면 됩니다.

    from agent.tools import TOOLS, call_tool  를 처음 import할 때
    통계 모델 6개가 동시에 초기화됩니다. 약 5~10초 소요됩니다.

    @st.cache_resource
    def load_agent(age_segment):
        return WMAgent(age_segment=age_segment)

  ② 프로젝트 루트에서 실행
    반드시 agent/ 폴더가 아닌 프로젝트 루트(agent/)에서 실행해야 합니다.
    cd C:\Users\user\Desktop\agent
    streamlit run ui/app.py

  ③ run_monte_carlo 단위
    모든 금액 단위는 만원입니다.
    3억원 → 30000, 250만원 → 250, 1억원 → 10000

  ④ API 비용
    claude-sonnet-4-6 기준 약 0.003달러/메시지.
    테스트 20개 전부 실행해도 약 0.06달러 수준.

  ⑤ depletion_prob 해석
    0.47 → "47%" → "100번 중 47번은 자산이 먼저 소진됨"
    UI에서 % 변환 후 직관적 표현 사용 권고.
